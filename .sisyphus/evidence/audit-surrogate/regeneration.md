# Canonical Regeneration Commands

## HonkVerifier.sol

`contracts/src/generated/HonkVerifier.sol` is auto-generated by the BB CLI from a compiled Noir circuit verification key.

### Steps

```bash
# 1. Compile the target circuit (e.g. aggregator_final)
cd circuits && nargo execute --package aggregator_final --prover-name Prover

# 2. Generate the verification key
bb write_vk --scheme ultra_honk -b circuits/aggregator_final/target/aggregator_final.json -o circuits/aggregator_final/target

# 3. Generate the Solidity verifier
bb write_solidity_verifier --scheme ultra_honk \
  -k circuits/aggregator_final/target/vk \
  -o contracts/src/generated/HonkVerifier.sol
```

Run all commands from the repository root.

Note: Until LatticeFold+ (Open Problem P2) is implemented, the generated verifier checks `keccak256(proof) == publicInputs[0]` as a structural stand-in.

## fhers.rs (pvthfhe-fhe crate)

`crates/pvthfhe-fhe/src/fhers.rs` is a hand-authored shim that delegates to `MockBackendInner`. It will be replaced when T33 wires the real `gnosisguild/fhe.rs` BFV API.

### Regeneration / wiring (T33)

1. Add `fhe` dependency to `crates/pvthfhe-fhe/Cargo.toml`
2. Replace each `// TODO(T33)` stub in `fhers.rs` with calls to the real `fhe::bfv` API
3. Run conformance tests: `cargo test -p pvthfhe-fhe`
