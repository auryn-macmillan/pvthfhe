## HIGH Severity Findings (5)

### HIGH-1: Empty Multi-Round Bootstrap Sigma Proof Verifies Successfully (NIZK/FORGE)

**File**: `crates/pvthfhe-nizk/src/bootstrap_sigma.rs:216-227`

`verify_multi` iterates over `proof.rounds`. When `rounds` is empty, the loop never executes, and the function returns `Ok(())`. An attacker submits `BootstrapSigmaMultiProof { rounds: vec![] }` and passes verification for any statement.

**Fix**: Add `if proof.rounds.is_empty() { return Err(...) }` at the top of `verify_multi`.

---

### HIGH-2: Zero-Step IVC Proof Bypass — Step Count Not Enforced (Nova/POISON)

**File**: `crates/pvthfhe-compressor/src/nova/mod.rs:1892-1898`

`NovaCompressor::prove_steps` only `tracing::debug!` logs when `steps.len() != self.ivc_steps` — execution continues to produce a RecursiveSNARK with zero fold iterations. `verify_steps` passes because initial state == final state is trivially satisfied.

**Fix**: Return `CompressorError::InvalidInput` when `steps.len() != self.ivc_steps`.

---

### HIGH-3: No Session Binding in Nova Step Circuits — Cross-Session Step Replay (Nova/REPLAY)

**Files**: `crates/pvthfhe-compressor/src/nova/mod.rs:1277-1279`, `fhe_compute_circuit.rs:1288-1293`

None of the 6 `StepCircuit` trait implementations bind session identifier in their R1CS constraints. The state vectors track computation accumulation (fold_count, step_count, hashes) but never include a session-aware domain separator. A step from session A can be substituted into session B's proof chain.

**Fix**: Seed initial state `z0[..]` with `hash(session_id || epoch_hash)` in every step circuit.

---

### HIGH-4: H2 Commit-Reveal Commitment Never Verified During DKG (DKG/POISON)

**File**: `crates/pvthfhe-aggregator/src/keygen/simulator.rs:269-295`

The Round 1 aggregator check validates duplicates and malformed NIZK but NEVER verifies the H2 commitment against `pk_i_hash` and `commitment_nonce`. An adversary can submit arbitrary commitment values and choose their public key after seeing honest parties' keys (classic rogue-key attack). C5 PoP provides a second layer of defense, but the H2 layer is completely inert.

**Fix**: Add commitment verification in the Round 1 check loop: `msg.commitment == compute_round1_commitment(party_id, session_id, &msg.pk_i_hash, &msg.commitment_nonce)`.

---

### HIGH-5: Four IvcBinding Fields Omitted From Statement Hash — Cross-Session Binding Gap (Solidity/FORGE)

**File**: `contracts/src/PvtFheVerifier.sol:558-580`

`shareVerificationHash`, `decryptNizkHash`, `dkgTranscriptHash`, `novaFinalStateCommitment` are validated as non-zero by `_requireIvcBindingValid` but NEVER included in the Poseidon statement hash via `_computeIvcStatementHash`. The `VerificationStatementV1.Statement` struct has no slots for them (FIELD_COUNT=19, no fields 20-23). An attacker can substitute arbitrary non-zero dummy values without changing the statement hash.

**Fix**: Add 4 new fields (20-23) to `VerificationStatementV1.Statement`, bump FIELD_COUNT to 23, PREIMAGE_LEN to 92 — or hash them into an existing placeholder field (`c6ProofSetRoot`/`cycloAccumulatorRoot`).

## MEDIUM Severity Findings (15)

### MEDIUM-1: PVSS Commitment Not Bound into T2 Fiat-Shamir Challenge (NIZK/REPLAY)
**File**: `crates/pvthfhe-nizk/src/sigma.rs:718-737`. `derive_challenge_from_commitment` omits `d_commitment`. Legacy path includes it but result is discarded. Fix: add `d_commitment` parameter.

### MEDIUM-2: Accumulator Verification Defers Full Fold Relation Check (NIZK/FORGE)
**File**: `crates/pvthfhe-nizk/src/adapter.rs:368-443`. `verify_accumulator_transcript` validates metadata, session_id, params_digest, norm bounds but never calls `verify_fold`. Acknowledged as intentional deferral to aggregator layer. Fix: rename to `validate_*` + distinct error variant.

### MEDIUM-3: Bootstrap Sigma Multi-Round Lacks Round Index Binding (NIZK/REPLAY)
**File**: `crates/pvthfhe-nizk/src/bootstrap_sigma.rs:152,199-211`. `prove_multi` always passes `round_index=0` to `derive_challenge`. A proof from round i can be moved to round j without detection. Fix: pass round index through.

### MEDIUM-4: BFV Sigma Lacks Rejection Sampling (NIZK/FORGE)
**File**: `crates/pvthfhe-nizk/src/bfv_sigma.rs:231-287`. Unlike `sigma.rs` (Lyubashevsky loop), BFV sigma produces proof in single pass with no rejection. Statistical ZK property weakened. Fix: add rejection sampling or document computational-ZK rationale.

### MEDIUM-5: BFV Sigma Challenge Derivation Has Ambiguous Encoding (NIZK/REPLAY)
**File**: `crates/pvthfhe-nizk/src/bfv_sigma.rs:407-471`. Variable-length fields (`session_id`, `binding_data`) concatenated without length prefixes in `derive_challenge`. Fix: use length-prefixed encoding.

### MEDIUM-6: Three StepCircuit Implementations Are Complete No-Ops (Nova/POISON)
**File**: `crates/pvthfhe-compressor/src/nova/mod.rs:353-362`. `DkgAggregationStepCircuit`, `KeyContributionStepCircuit`, `PkAggregationStepCircuit` all `synthesize()` as `Ok(z.to_vec())` — zero R1CS constraints, pure pass-through. Fix: implement real constraints or error on usage.

### MEDIUM-7: FheComputeStepCircuit Idle Path Accepts Empty-Witness Steps (Nova/POISON)
**File**: `crates/pvthfhe-compressor/src/nova/fhe_compute_circuit.rs:1353-1363`. When `!has_data`, passes state through unchanged + increments step_count. N idle steps produce valid proof with no computation. Fix: return `SynthesisError::Unsatisfiable` or freeze step_count.

### MEDIUM-8: Duplicate/Out-of-Order Steps Not Enforced at Nova Level (Nova/POISON)
**File**: `crates/pvthfhe-compressor/src/nova/fhe_compute_circuit.rs:1339-1346`. Step counter wraps modulo `data.len()` without bounds check. counter from previous run can carry over into new prove call. Fix: add `raw_step >= data.len() → SynthesisError::Unsatisfiable`.

### MEDIUM-9: Thread-Local State Cross-Session Reuse (Nova/REPLAY)
**File**: `crates/pvthfhe-compressor/src/nova/mod.rs:1882-1898`. Only `CYCLO_FOLD_STEP_COUNTER` is reset before `prove_steps`. Other counters (FHE, Ajtai, ShareVerify, SchemeSwitch) persist across calls. Fix: reset ALL counters at prove entry.

### MEDIUM-10: `prove()` Single-Step Path Bypasses `ivc_steps` Enforcement (Nova/POISON)
**File**: `crates/pvthfhe-compressor/src/nova/mod.rs:2164-2175`. `prove()` always uses `steps.len()=1` regardless of `ivc_steps`. Compressor with `ivc_steps=100` produces/verifies a 1-step proof. Fix: enforce `ivc_steps == 1` in single-step path or route through `prove_steps`.

### MEDIUM-11: `contextId` Hardcoded to `bytes32(0)` (Solidity/FRONT-RUN)
**File**: `contracts/src/PvtFheVerifier.sol:561`. Statement hash lacks execution context disambiguation. Fix: populate from protocol label + parent session when Phase 2 seam closes.

### MEDIUM-12: Missing Adversarial Tests for Cross-Session Proof Replay (Solidity/FORGE)
**Files**: `contracts/test/PvtFheVerifier.t.sol`, `IvcDeciderWiring.t.sol`. No tests for: wrong c5ProofRoot with valid proof, cross-session IVC binding, mutated shareVerificationHash/decryptNizkHash/dkgTranscriptHash/novaFinalStateCommitment. Existing mock test helpers replicate the same 4-field omission. Fix: add RED tests.

### MEDIUM-13: `aggregate_decrypt` Backend Layer Ignores Session Identifier (DKG/POISON)
**File**: `crates/pvthfhe-fhe/src/fhers.rs:1363`. `aggregate_decrypt` accepts `_session_id: &[u8]` but ignores it (underscore-prefixed). Fix: verify session binding in backend layer.

### MEDIUM-14: `encode_fields` Panics on Oversized Fields (DKG/HALT)
**File**: `crates/pvthfhe-fhe/src/wire.rs:173-188`. `.expect()` on `u32::try_from(field.len())` panics and crashes process if field > u32::MAX bytes. Fix: return `WireError` instead.

### MEDIUM-15: KeygenShareV1 and PublicKeyV1 Have No Field Size Bounds (DKG/HALT)
**File**: `crates/pvthfhe-fhe/src/wire.rs:159-224`. Only `DecryptShareV2` has `MAX_DECRYPT_SHARE_BYTES`. Keygen/PublicKey share types have no maximum size, vulnerable to memory exhaustion. Fix: add `MAX_KEYGEN_SHARE_BYTES`, `MAX_PUBLIC_KEY_BYTES`.
