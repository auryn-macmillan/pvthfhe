## Classification rule

- **PROVED-WITH-CITATION**: proof file contains a direct argument matching the current implemented/specified claim.
- **PROVED-VACUOUS**: proof file discharges only a surrogate, deferred, conditional, N/A, or engineering-target version of the obligation under the current code path.
- **GAP**: proof file itself leaves the obligation open for the implemented code.
- **MISSING**: no proof document exists.

## Key findings

- **Unique GAP/MISSING theorem:** `P2-T4` is the only `GAP`. Its own proof file states that accumulator binding is conditional on two unimplemented obligations: arithmetic norm enforcement in `validate_witness` and replacement of the SHA-256 hash-chain surrogate with a linear lattice commitment.
- **19 vs 20 discrepancy resolved:** `P1-T4` is present in `docs/security-proofs/obligations.md` and `docs/security-proofs/p1/T4.md`, but it is absent from both `paper/claims-table.md` and `paper/main.tex`. That is the missing 20th paper theorem.
- `paper/main.tex` contains **19** `\begin{theorem}` environments.
- `docs/security-proofs/obligations.md` currently contains **0** audit classifications (`PROVED-WITH-CITATION|PROVED-VACUOUS|GAP|MISSING`), so this audit lives here rather than in-place.

## Discrepancy resolution details

### Obligation present but absent from paper

- `P1-T4` appears in `docs/security-proofs/obligations.md` and has a proof file at `docs/security-proofs/p1/T4.md`.
- `paper/claims-table.md` lists `P1-T1`, `P1-T2`, `P1-T3`, and `P1-T5`, but not `P1-T4`.
- `paper/main.tex` likewise includes theorem environments for `P1-T1`, `P1-T2`, `P1-T3`, and `P1-T5`, but no theorem environment for `P1-T4`.

### Why `P2-T4` is the unique GAP

`docs/security-proofs/p2/T4.md` expressly conditions accumulator binding on two things the current code does not do:

1. arithmetic norm enforcement in `validate_witness`; and
2. replacing the current SHA-256 hash-chain surrogate with a linear algebraic accumulator commitment.

Those are not documentation caveats only; they are implementation obligations explicitly marked “NOT IMPLEMENTED,” so the theorem is not discharged for the repository as it exists.
