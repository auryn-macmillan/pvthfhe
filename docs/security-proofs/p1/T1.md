# P1-T1 — Completeness of the Frozen SLAP-Style Proof

## Theorem Statement

Let

\[
x=(\mathsf{session\_id},\mathsf{participant\_id},\mathsf{ciphertext\_bytes},\mathsf{decrypt\_share\_bytes},\mathsf{pvss\_commitment},q,N,B_e)
\]

be a public statement accepted by `RealNizkAdapter::validate_statement`, and let

\[
w=(s,\mathbf e,\rho)
\]

be a witness such that:

1. `pvss_commitment = SHA256(session_id || participant_id_le || s_be)`;
2. every coefficient of \(\mathbf e\) satisfies \(|e_j| \le B_e\);
3. `len(e) > 0`;
4. the prover samples `y_s \leftarrow [0,q)` and `y_{e,j} \leftarrow [-B_e,B_e]` from the deterministic seed `SHA256(statement_bytes || witness_bytes)`;
5. the Fiat–Shamir challenge bytes are the first 16 bytes of `SHA256(session_id || pvss_commitment || t_bytes || statement_bytes)`, and `c \in \{-1,0,1\}` is derived from byte 15 of that 16-byte array modulo 3; and
6. the prover sets

\[
z_s \equiv y_s + c s \pmod q,
\qquad
z_{e,j} = y_{e,j} + c e_j.
\]

Then the proof produced by `RealNizkAdapter::prove` is accepted by `RealNizkAdapter::verify` with probability exactly \(1\) in the mathematical model, and with probability at least

\[
1-\varepsilon_{\mathrm{enc}}-\varepsilon_{\mathrm{parse}}
\]

in the concrete implementation, where the only bad events are malformed serialization or external implementation faults excluded by the frozen interface.

## Proof Strategy

Direct algebraic correctness. Every verifier predicate is recomputed from the same opened witness values and the same canonical transcript bytes used by the honest prover, so each check succeeds identically.

## Reduction

There is no hardness reduction for completeness, but the verifier still checks a conjunction of predicates that must each be discharged explicitly.

1. **Statement well-formedness.** `validate_statement` enforces `q > 0`, `N > 0`, non-empty `session_id`, and non-empty ciphertext/decrypt-share byte strings. Honest inputs therefore pass the syntactic gate.
2. **Commitment check.** The proof opens `secret_share_open = s`. The verifier recomputes

   \[
   \widehat C = \mathrm{SHA256}(\mathsf{session\_id} \| \mathsf{participant\_id}_{\mathrm{le}} \| s_{\mathrm{be}}),
   \]

   which equals the statement commitment by premise.
3. **Opened-error bound.** By premise, every opened coefficient satisfies \(|e_j| \le B_e\), so `coeffs_within_bound(error_open, B_e)` returns true.
4. **Response bound.** Because the mask sampler outputs \(|y_{e,j}| \le B_e\) and the challenge satisfies \(c \in \{-1,0,1\}\),

   \[
   |z_{e,j}| = |y_{e,j} + c e_j| \le |y_{e,j}| + |e_j| \le 2 B_e.
   \]

   Hence the verifier's abort-threshold check `|z_e| <= 2*B_e` always passes for honest proofs.
5. **Challenge recomputation.** The prover and verifier hash the same byte string `session_id || pvss_commitment || t_bytes || statement_bytes`, truncate the SHA-256 digest to its first 16 bytes exactly as `challenge_bytes` does, and derive the same ternary challenge weight from byte 15 modulo 3.
6. **Sigma consistency.** The verifier recovers

   \[
   \widehat y_s \equiv z_s - c s \pmod q,
   \qquad
   \widehat y_{e,j} = z_{e,j} - c e_j,
   \]

   which are exactly the honest masks \((y_s, y_{e,j})\). Re-encoding those masks therefore reproduces the original `t_bytes`.
7. **Conclusion.** Every verifier predicate holds simultaneously, so the honest proof is accepted.

## Parameter Constraints

- `q >= 2`; concretely the checked prototype uses `q = 65537` in default tests.
- `N >= 1`; concretely the checked prototype uses `N = 1024` in default tests.
- `B_e >= 0`; concretely the checked prototype uses `B_e = 17` in default tests.
- `participant_id` is encoded as two little-endian bytes inside the commitment preimage, but as two big-endian bytes inside `statement_bytes` exactly as `RealNizkAdapter::statement_bytes` serializes the public statement.
- `secret_share` is encoded as exactly eight big-endian bytes.
- `len(error_open) = len(z_e) > 0`.
- The challenge space is ternary: `c in {-1,0,1}`.
- Honest completeness requires the verifier threshold `|z_{e,j}| <= 2 B_e`; this is exactly the bound induced by the mask sampler and ternary challenge.
- Canonical serialization of `statement_bytes` and `t_bytes` is required so prover and verifier hash identical inputs; concretely `statement_bytes` uses length-prefixed `session_id`, big-endian `participant_id`, length-prefixed ciphertext/decrypt-share byte arrays, the 32-byte commitment digest, and big-endian encodings of `(q, N, B_e)`.

## Tightness

The theorem is tight: there is no reduction loss. Quantitatively,

\[
\Pr[\mathsf{Verify}(x,\mathsf{Prove}(x,w))=1] = 1
\]

in the idealized arithmetic model. Any concrete loss comes only from serialization or parser failure external to the algebraic proof argument.
