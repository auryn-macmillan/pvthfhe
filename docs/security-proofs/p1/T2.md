# P1-T2 — Soundness and Extraction for the Implemented Proof Relation

## Theorem Statement

Let `Verify` be `RealNizkAdapter::verify`. For every PPT adversary \(\mathcal A\) that outputs a statement-proof pair \((x,\pi)\) accepted by `Verify`, there exists a rewinding extractor \(\mathcal E\) running in time linear in `|proof_bytes|` such that, except with probability at most the SHA-256 binding advantage,

\[
\mathcal E(x,\pi)=(s,\mathbf e,\rho)
\]

and the extracted witness satisfies:

1. `pvss_commitment = SHA256(session_id || participant_id_le || s_be)`;
2. every coefficient of \(\mathbf e\) satisfies \(|e_j| \le B_e\);
3. the published response vector satisfies \(|z_{e,j}| \le 2 B_e\);
4. the transcript commitment `t_bytes` is exactly the encoding of the recovered masks

   \[
   y_s \equiv z_s - c s \pmod q,
   \qquad
   y_{e,j} = z_{e,j} - c e_j;
   \]

5. the challenge `c` is the Fiat–Shamir value derived by truncating `SHA256(session_id || pvss_commitment || t_bytes || statement_bytes)` to its first 16 bytes and mapping byte 15 modulo 3 into `{-1,0,1}`.

Equivalently: any accepted proof already contains an extractable witness for the implemented P1 relation, and any false acceptance would imply a break of the pvss commitment binding or of canonical transcript hashing.

## Proof Strategy

The serialized proof contains only the masked sigma transcript (t_bytes, z_s, z_e);
no witness openings are present in the proof payload (confirmed at 90d6787).
Extraction therefore requires a rewinding argument: we fork the prover on the
Fiat--Shamir challenge to obtain two accepting transcripts with the same first
message but different challenges. The challenge difference Δ = c - c' is non-zero
(forking lemma) and invertible in the Cyclo commitment ring under Lemma 9
(φ_commit = 256, q_commit ≈ 2⁵⁰), yielding the witness via (z - z') · Δ⁻¹.
The reduction targets are the forking lemma (standard ROM), Lemma 9 invertibility
(accepted assumption), and SHA-256 binding.

## Reduction

Define the rewinding extractor E (ROM, forking lemma) as follows.

1. **First pass.** Run the prover P with random oracle H. Receive an accepting
   transcript (t_bytes, z_s, z_e, c) where the challenge c is derived via
   Fiat--Shamir:
   c = trunc_16(H(session_id || commitment || t_bytes || statement_bytes))
   with byte 15 mapped modulo 3 into {-1, 0, 1}.

2. **Rewind.** Rewind P to the point after t_bytes is fixed (i.e., after P has
   committed to its first message). Provide a fresh random oracle H' (independent
   of H). Receive a second accepting transcript (t_bytes, z_s', z_e', c').

3. **Forking success.** If c' = c, abort and repeat from step 2. By the forking
   lemma, a successful fork (c' ≠ c) occurs with probability at least
   ε - ε²/|C| after one rewind, where |C| = 3 for the ternary challenge set.

4. **Challenge difference.** Compute Δ = c - c'. Since c' ≠ c and the challenge
   set is {-1, 0, 1}, Δ ∈ {±1, ±2}. Under Lemma 9
   (docs/security-proofs/lemma9.md, §0), Δ is invertible in the Cyclo commitment
   ring (φ_commit = 256, q_commit ≈ 2⁵⁰) except with negligible probability η.

5. **Extract secret share.**
   z_s = y_s + c·s  and  z_s' = y_s + c'·s
   ⇒  z_s - z_s' = (c - c')·s = Δ·s
   ⇒  s = (z_s - z_s') · Δ⁻¹

6. **Extract error vector.** For each coefficient j:
   z_{e,j} = y_{e,j} + c·e_j  and  z_{e,j}' = y_{e,j} + c'·e_j
   ⇒  z_{e,j} - z_{e,j}' = Δ·e_j
   ⇒  e_j = (z_{e,j} - z_{e,j}') · Δ⁻¹

7. **Recover masks.**
   y_s = z_s - c·s
   y_{e,j} = z_{e,j} - c·e_j   for each j

8. **Verify the extracted witness.**
   (a) |s| ≤ B_S / |Δ| (norm bound, Lemma 9 inverse bound)
   (b) |e_j| ≤ B_e for all j
   (c) SHA256(session_id || participant_id_le || s_be) == commitment
   (d) t_bytes encodes (y_s, y_e) correctly under the canonical encoding

Now analyze any accepting output of A under the extractor.

1. **Parsing.** Acceptance implies the proof bytes decode to the masked transcript
   (t_bytes, z_s, z_e). The extractor acquires two such transcripts from separate
   oracle queries; both decode successfully by the acceptance condition.

2. **Length consistency.** Both transcripts share the same t_bytes and therefore
   the same N = len(z_e) = len(z_e'). The extracted error vector is well-defined
   coefficientwise.

3. **Commitment opening.** Step 8(c) verifies SHA-256 binding. If the extracted s
   fails this check while both transcripts were accepted, then A produced an
   accepting proof with a commitment that binds to a different value than the
    extracted s, a SHA-256 binding break.

4. **Bounded witness.** Step 8(a--b) verifies norm bounds on s and e. If the
   extraction succeeds (Δ invertible) but the bounds fail, then A produced an
   accepting transcript pair whose masked responses would imply a witness outside
    the admissible range, contradicting the verification predicate that checks
   |z_{e,j}| ≤ 2 B_e on each accepted transcript individually.

5. **Transcript consistency.** Step 8(d) verifies that t_bytes encodes the
   recovered masks. If this fails while both transcripts were individually accepted
   (meaning each transcript's own t_bytes matched its own masks), then A produced
    two transcripts with the same t_bytes but inconsistent masks, implying the
   mask encoding is not a function of the transcript alone, which contradicts the
   deterministic encoding.

6. **Reduction to binding failure.** Suppose A outputs an accepted proof for a
   statement with no valid witness. From such an adversary, construct B that:
   (a) Uses E to extract a candidate witness (s, e, ρ) via rewinding.
   (b) If extraction succeeds but the witness fails verification step 8(c), output
       (s, s') as a SHA-256 collision for the commitment prefix.
   This contradicts the assumed binding of SHA-256 on the commitment domain.

7. **Conclusion.** Under the ROM, Lemma 9, and SHA-256 collision resistance, any
   accepting proof yields an extracted witness via rewinding. The extraction
   operates on the masked sigma transcript; no witness openings are required in
   the proof payload.

## Parameter Constraints

- `q > 0`, so modular subtraction for `y_s` is well-defined.
- `N > 0`; it is public metadata bound into `statement_bytes` and therefore into the challenge hash.
- `B_e >= 0`.
- `len(error_open) = len(z_e) > 0`.
- Challenge derivation uses the first 16 bytes of SHA-256 output and maps byte 15 of that truncated array modulo 3 into `{-1,0,1}`.
- The commitment preimage format is fixed to `session_id || participant_id_le || secret_share_be`, while `statement_bytes` separately encodes `participant_id` as big-endian inside the Fiat–Shamir hash input; changing either encoding changes the theorem.
- The theorem is scoped to the implemented verifier relation; it does **not** claim soundness of any omitted ciphertext/decrypt-share arithmetic that the current verifier does not check.

## Tightness

Let ε be the adversary's acceptance probability (probability that P produces an
accepting transcript on the first query). Let η = Pr_Δ[Lemma 9 invertibility
fails for challenge difference Δ]. Then:

  Pr[E outputs a valid witness] ≥ ε² - ε/|C| - η - Adv_bind^SHA256

where:
  - ε² is the standard forking-lemma loss: the extractor must succeed at the
    first oracle query AND on the rewound query;
  - ε/|C| accounts for the probability that c' = c on rewind, where
    |C| = 3 for the ternary challenge set {-1, 0, 1};
  - η is the Lemma 9 failure probability, assumed negligible under the
    accepted assumption (docs/security-proofs/lemma9.md, §0);
  - Adv_bind^SHA256 is the commitment binding advantage.

The forking-lemma loss ε² is standard for ROM rewinding extractors. The ε/|C|
term reflects the ternary challenge space. No additional loss arises from the
mask extraction step, since Lemma 9 guarantees Δ⁻¹ exists with overwhelming
probability and the extraction arithmetic is exact.
