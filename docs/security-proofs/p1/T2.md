# P1-T2 — Soundness and Extraction for the Implemented Proof Relation

## Theorem Statement

Let `Verify` be `RealNizkAdapter::verify`. For every PPT adversary \(\mathcal A\) that outputs a statement-proof pair \((x,\pi)\) accepted by `Verify`, there exists a straight-line extractor \(\mathcal E\) running in time linear in `|proof_bytes|` such that, except with probability at most the SHA-256 binding advantage,

\[
\mathcal E(x,\pi)=(s,\mathbf e,\rho)
\]

and the extracted witness satisfies:

1. `pvss_commitment = SHA256(session_id || participant_id_le || s_be)`;
2. every coefficient of \(\mathbf e\) satisfies \(|e_j| \le B_e\);
3. the published response vector satisfies \(|z_{e,j}| \le 2 B_e\);
4. the transcript commitment `t_bytes` is exactly the encoding of the recovered masks

   \[
   y_s \equiv z_s - c s \pmod q,
   \qquad
   y_{e,j} = z_{e,j} - c e_j;
   \]

5. the challenge `c` is the Fiat–Shamir value derived by truncating `SHA256(session_id || pvss_commitment || t_bytes || statement_bytes)` to its first 16 bytes and mapping byte 15 modulo 3 into `{-1,0,1}`.

Equivalently: any accepted proof already contains an extractable witness for the implemented P1 relation, and any false acceptance would imply a break of the pvss commitment binding or of canonical transcript hashing.

## Proof Strategy

Use the fact that the proof payload opens the witness directly. Unlike the abstract B.D.3 skeleton, the checked implementation does not need rewinding or a forking lemma: extraction is obtained by decoding the proof and re-running the verifier's own checks. The only nontrivial reduction target is SHA-256 binding for `pvss_commitment`.

## Reduction

Define the extractor \(\mathcal E\) as follows.

1. Parse `proof_bytes` as `ProofPayload`.
2. Output `(secret_share_open, error_open, randomness_open)`.
3. Recompute every verifier predicate.

Now analyze any accepting output of \(\mathcal A\).

1. **Successful parsing.** Acceptance implies `ProofPayload::decode` succeeded, so the extractor obtains concrete values `t_bytes`, `z_s`, `z_e`, `secret_share_open`, `error_open`, and `randomness_open`.
2. **Length consistency.** Acceptance implies `len(z_e) = len(error_open)`, so the extracted error vector is well-defined coefficientwise.
3. **Commitment opening.** Acceptance implies

   \[
   \mathrm{SHA256}(\mathsf{session\_id} \| \mathsf{participant\_id}_{\mathrm{le}} \| s_{\mathrm{be}})=\mathsf{pvss\_commitment}.
   \]

   Therefore the extracted `s` is a valid opening of the public commitment.
4. **Bounded witness.** Acceptance implies \(|e_j| \le B_e\) for every extracted coefficient and \(|z_{e,j}| \le 2B_e\) for every response coefficient.
5. **Transcript consistency.** The verifier recomputes the challenge hash from `(session_id, pvss_commitment, t_bytes, statement_bytes)`, truncates the SHA-256 digest to 16 bytes, derives `c in {-1,0,1}` from byte 15 modulo 3, reconstructs the masks `(y_s, y_e)`, re-encodes them, and checks equality with `t_bytes`. Acceptance therefore certifies that the extracted witness is internally consistent with the published sigma transcript.
6. **Reduction to binding failure.** Suppose, for contradiction, that \(\mathcal A\) outputs an accepted proof for a statement with no valid witness. Because steps 1–5 recover a concrete witness satisfying every verifier predicate, the only remaining possibility is that the public commitment admits two distinct openings. From such an adversary, construct \(\mathcal B\) that outputs a SHA-256 collision/second-preimage pair for the fixed prefix `session_id || participant_id_le`:

   \[
   (s,s') \text{ with } s \neq s' \text{ and } H(\mathsf{prefix}\|s_{\mathrm{be}})=H(\mathsf{prefix}\|s'_{\mathrm{be}}).
   \]

   This contradicts the assumed binding of SHA-256 on the commitment domain.
7. **Conclusion.** Any accepting proof yields an extracted witness unless the adversary breaks the commitment binding assumption. No Module-LWE, Module-SIS, rewinding, or ROM forking loss is needed for the implemented extractor because the witness is carried explicitly in the proof payload.

## Parameter Constraints

- `q > 0`, so modular subtraction for `y_s` is well-defined.
- `N > 0`; it is public metadata bound into `statement_bytes` and therefore into the challenge hash.
- `B_e >= 0`.
- `len(error_open) = len(z_e) > 0`.
- Challenge derivation uses the first 16 bytes of SHA-256 output and maps byte 15 of that truncated array modulo 3 into `{-1,0,1}`.
- The commitment preimage format is fixed to `session_id || participant_id_le || secret_share_be`, while `statement_bytes` separately encodes `participant_id` as big-endian inside the Fiat–Shamir hash input; changing either encoding changes the theorem.
- The theorem is scoped to the implemented verifier relation; it does **not** claim soundness of any omitted ciphertext/decrypt-share arithmetic that the current verifier does not check.

## Tightness

The reduction is essentially tight. If \(\epsilon\) is the adversary's acceptance probability and `Adv_bind^SHA256` is the best SHA-256 binding advantage on the fixed commitment domain, then

\[
\Pr[\mathcal E(x,\pi) \text{ outputs a valid witness}] \ge \epsilon - \mathrm{Adv}_{\mathrm{bind}}^{\mathrm{SHA256}}.
\]

There is no forking loss, no quadratic ROM loss, and no extractor abort term beyond proof decoding failure already ruled out by acceptance.
