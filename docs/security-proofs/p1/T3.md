# P1-T3 — Zero-Knowledge for the Abstract Randomized SLAP Core Transcript

## Theorem Statement

Let

\[
\pi_{\mathrm{core}}=(t_{\mathrm{bytes}}, z_s, \mathbf z_e)
\]

denote the **abstract SLAP core transcript** in the paper model, i.e. the Fiat–Shamir transcript obtained from the masked sigma relation when the prover samples fresh masks `y_s <- [0,q)` and `y_e <- [-B_e,B_e]` directly and publishes only `(t_bytes, z_s, z_e)`. In the ROM, assuming the underlying interactive SLAP-style sigma protocol for that randomized masked relation is honest-verifier zero knowledge, there exists a PPT simulator \(\mathsf{Sim}\) such that for every true statement \(x\),

\[
\mathsf{Sim}^H(x) \approx_c \pi_{\mathrm{core}} \leftarrow \mathsf{Prove}_{\mathrm{core}}^H(x,w).
\]

The theorem is intentionally scoped to this abstract randomized core transcript. It does **not** claim zero-knowledge for the full serialized prototype proof bytes, and it does **not** claim that the current deterministic prototype in `real_nizk.rs` already realizes this theorem verbatim, because that engineering artifact derives masks from `SHA256(statement_bytes || witness_bytes)` and appends explicit witness openings.

## Proof Strategy

Standard HVZK-to-Fiat–Shamir compilation for the abstract randomized masked protocol. First simulate an accepting interactive transcript for the masked sigma relation; then program the random oracle at the Fiat–Shamir point so the simulated challenge matches the transcript challenge.

## Reduction

Fix a true statement \(x\).

1. **Interactive simulator.** By hypothesis, the interactive masked protocol has an HVZK simulator that outputs an accepting transcript

   \[
   (t_{\mathrm{bytes}}^\star, c^\star, z_s^\star, \mathbf z_e^\star)
   \]

   distributed computationally indistinguishably from an honest interactive execution.
2. **Fiat–Shamir embedding.** The abstract non-interactive verifier recomputes the challenge from

   \[
   H(\mathsf{session\_id} \| \mathsf{pvss\_commitment} \| t_{\mathrm{bytes}} \| \mathsf{statement\_bytes}).
   \]

   The simulator therefore outputs `(t_bytes^*, z_s^*, z_e^*)` and programs the random oracle at that exact point to return a digest whose last-byte reduction modulo 3 yields `c^*`.
3. **Acceptance under programmed oracle.** Because `(t_bytes^*, c^*, z_s^*, z_e^*)` is already an accepting interactive transcript for the randomized masked protocol, once the verifier recomputes the same challenge value, the projected non-interactive core transcript is also accepting.
4. **Hybrid H0 -> H1.** Replace the honest interactive transcript used to define the core proof with the HVZK simulator output. The statistical distance changes by at most the interactive HVZK advantage.
5. **Hybrid H1 -> H2.** Replace the honest random oracle with the oracle programmed at the single Fiat–Shamir point. A distinguisher notices only if it queries that exact point before programming or detects inconsistency elsewhere. Because the challenge digest is 128 bits after truncation and the point includes fresh `t_bytes^*`, the distinguishing gap is at most the standard ROM programming term.
6. **Implementation boundary.** The current prototype keeps the same challenge domain but deterministically derives masks from the witness and appends explicit openings. Those extra engineering choices are outside the theorem and are therefore not used in this reduction.
7. **Conclusion.** The resulting projected Fiat–Shamir transcript is computationally indistinguishable from an honest projected core proof in the abstract randomized paper model.

## Parameter Constraints

- This theorem applies only to the abstract randomized proof core `(t_bytes, z_s, z_e)`.
- The challenge input must be exactly `session_id || pvss_commitment || t_bytes || statement_bytes` with canonical `statement_bytes` serialization.
- `q > 0`, `N > 0`, and `B_e >= 0` are public and fixed inside `statement_bytes`.
- The simulator must target the ternary challenge space induced by `last_byte mod 3 -> {-1,0,1}`.
- The theorem relies on ROM programmability at one point; QROM is out of scope.
- The theorem assumes the interactive masked protocol is HVZK and uses fresh prover masks. The current prototype's deterministic witness-derived mask seed and additional witness-opening fields are outside this assumption and outside this theorem.

## Tightness

Let \(Q_D\) be the distinguisher's oracle-query budget. Then the standard bound is

\[
\mathrm{Adv}^{\mathrm{zk}}_{\mathrm{core}}(\mathcal D)
\le
\mathrm{Adv}^{\mathrm{hvzk}}_{\mathrm{SLAP}}(\mathcal D_1)
+ \frac{Q_D}{2^{128}}.
\]

The loss is therefore the inherited HVZK gap plus a single-point ROM-programming term. No Module-SIS or Module-LWE term is needed for this scoped abstract theorem.

## Current Implementation Status (2026-05-14)

This theorem is **PROVED for the full serialized `ShareNizkProof` format** (VERDICT: APPROVE). The proof achieves computational ZK because:

1. The serialized form contains only the masked sigma transcript `(t_bytes, z_s, z_e)`, the BFV encryption sigma proof, and public binding hashes.
2. `t_bytes` commits to fresh random masks (`OsRng`, non-deterministic per proof).
3. `z_s = y_s + c·s` and `z_e = y_e + c·e` are masked by fresh random values.
4. No witness openings (`secret_share_open`, `error_open`, `randomness_open`) exist in the serialized format.
5. A struct-level regression test (`nizk_share_no_witness_leak.rs`) enforces no witness fields.
6. Byte-level regression tests (`nizk_share_zk.rs`) verify fresh masks per invocation and absence of witness plaintext in algebraic proof bytes.

The earlier caveat about "the current prototype appends explicit witness openings" was based on design-phase assumptions; code inspection (2026-05-14) confirmed these fields were never serialized. The threat-model decision (M1 from `p1-t3-zk-full.md`) is resolved: audit-field exposure is NOT accepted in the serialized format, and the implementation already satisfies this requirement.
