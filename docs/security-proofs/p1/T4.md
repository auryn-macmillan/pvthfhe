# P1-T4 — Simulation-Extractability is Deferred

## Theorem Statement

Under the frozen P1 threat model, simulation-extractability is **not** a required baseline theorem. The baseline P1 package stops at:

1. completeness for honest proofs;
2. extraction from accepted prover-generated proofs; and
3. ROM simulation for the scoped SLAP core transcript.

If a future P2 interface gives an adversary access to simulated accepting P1 proofs before it produces a fresh proof, then a stronger theorem must be added for that upgraded setting. That stronger theorem is not claimed here.

## Proof Strategy

Dependency analysis against the frozen threat model and interface. Show that the current sequential P1->P2 composition never creates the simulator-oracle attack surface that would make simulation-extractability necessary.

## Reduction

1. **Threat-model baseline.** `.sisyphus/research/p1/threat-model.md` fixes the baseline to static corruption, ROM, and standard proof-of-knowledge with rewinding/knowledge extraction. It explicitly marks simulation-soundness and straight-line extraction as non-baseline properties.
2. **Composition boundary.** `.sisyphus/design/p1/interface-spec.md` exports only verifier-consumable statements and proofs. It does not export a simulator oracle or any API by which an adversary first receives simulated accepting proofs and then outputs fresh ones.
3. **Why T2 + T3 are sufficient.** In the frozen setting, accepted proofs that matter downstream are prover-generated proofs consumed by verification and later folding. For those objects, completeness and extraction are the required guarantees; seeing simulated proofs is not part of the experiment.
4. **Why plain FS does not imply sim-ext automatically.** Even though the core transcript admits ROM simulation, that does not upgrade ordinary extraction into extraction relative to simulator access. A future theorem would require a different experiment and likely a stronger transform.
5. **Future upgrade target.** If the interface changes, the upgraded reduction should use transcript tagging or another simulation-extractable compilation so that any fresh accepting proof after simulator access still yields a witness, except with explicitly bounded transform loss plus any inherited binding failure.
6. **Conclusion.** The correct paper claim today is deferral, not proof.

## Parameter Constraints

- Baseline model: static corruption, malicious PPT adversary, ROM, rewinding extractor.
- No simulator oracle is available to the adversary in the frozen P1->P2 composition.
- Any future upgrade must preserve the exact public parameter binding `(session_id, participant_id, q, N, B_e)` and the same SHA-256 commitment preimage format.
- Any future quantitative theorem must publish its own transform-specific loss; no such loss is silently inherited from T2 or T3.

## Tightness

No tightness claim is made because no simulation-extractability theorem is proved. Relative to the frozen baseline experiment, the additional loss is exactly zero because the property is out of scope rather than assumed.

## OUT OF SCOPE: standard PoK suffices per threat model

The frozen P1 threat model and interface do not require adversarial robustness against reuse of simulated accepting transcripts. Accordingly, T4 is recorded as **DEFERRED**, not **PROVED**.
