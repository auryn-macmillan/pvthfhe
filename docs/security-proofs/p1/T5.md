# P1-T5 — Binding of `pvss_commitment` for the B.I.4 Proof Package

## Theorem Statement

For the B.I.4 proof package, the task-frozen T5 obligation is the implementation-level binding theorem for `pvss_commitment`. This replaces the earlier research-stage batch-soundness placeholder from the skeleton inventory and is the theorem exported by the completed paper bundle for the current direct-opening verifier.

Let

\[
C = \mathrm{SHA256}(\mathsf{session\_id} \| \mathsf{participant\_id}_{\mathrm{le}} \| s_{\mathrm{be}})
\]

be the commitment function implemented by `RealNizkAdapter::commitment_hash`. For every PPT adversary \(\mathcal A\), if \(\mathcal A\) outputs two distinct openings \(s \ne s'\) such that

\[
\mathrm{SHA256}(\mathsf{session\_id} \| \mathsf{participant\_id}_{\mathrm{le}} \| s_{\mathrm{be}})
=
\mathrm{SHA256}(\mathsf{session\_id} \| \mathsf{participant\_id}_{\mathrm{le}} \| s'_{\mathrm{be}}),
\]

then one obtains a SHA-256 collision on the concrete commitment domain. Therefore `pvss_commitment` is computationally binding under SHA-256 collision resistance.

Equivalently, except with negligible probability, an accepted proof binds the public commitment to a unique `secret_share_open` value.

## Proof Strategy

Direct reduction. Any successful double opening immediately yields a collision for SHA-256 on two distinct byte strings with the same fixed prefix.

## Reduction

Construct a collision-finding adversary \(\mathcal B\) against SHA-256 from any commitment double-opener \(\mathcal A\).

1. \(\mathcal B\) runs \(\mathcal A\) on public inputs `(session_id, participant_id)`.
2. \(\mathcal A\) outputs two openings `s` and `s'` with `s != s'` and equal commitment digest.
3. \(\mathcal B\) forms the two full hash preimages

   \[
   m = \mathsf{session\_id} \| \mathsf{participant\_id}_{\mathrm{le}} \| s_{\mathrm{be}},
   \qquad
   m' = \mathsf{session\_id} \| \mathsf{participant\_id}_{\mathrm{le}} \| s'_{\mathrm{be}}.
   \]

4. Because `s != s'` and both are encoded as fixed-width 8-byte big-endian integers, `m != m'`.
5. By assumption on \(\mathcal A\), `SHA256(m) = SHA256(m')`. Thus \(\mathcal B\) outputs a valid SHA-256 collision.
6. Therefore any efficient double opening breaks SHA-256 collision resistance, so no such efficient double opening exists under the assumption.

For accepted proofs, the verifier explicitly recomputes the commitment from `secret_share_open`. Hence any two accepted proofs for the same statement with different opened secrets would instantiate the same collision reduction.

## Parameter Constraints

- `session_id` is encoded as raw UTF-8 bytes exactly as stored in the statement.
- `participant_id` is encoded as exactly two little-endian bytes.
- `secret_share` is encoded as exactly eight big-endian bytes.
- The digest function is SHA-256 with 32-byte output.
- Binding is claimed on the concrete implemented domain above; changing byte order or width changes the theorem.
- The theorem is computational, not information-theoretic.

## Tightness

The reduction is tight up to constant overhead:

\[
\mathrm{Adv}^{\mathrm{bind}}_{\mathrm{pvss}}(\mathcal A)
\le
\mathrm{Adv}^{\mathrm{coll}}_{\mathrm{SHA256}}(\mathcal B).
\]

\(\mathcal B\) makes one call to \(\mathcal A\) and performs only linear-time encoding work. There is no forking loss, no hybrid cascade, and no additional assumption beyond collision resistance on the stated domain.
